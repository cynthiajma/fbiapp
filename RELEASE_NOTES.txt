================================================================================
LIMITATIONS
================================================================================

PLATFORM LIMITATIONS
--------------------
1. Web Platform
   - Audio playback uses data URLs (may have size limitations)
   - CSV export uses browser download functionality
   - Limited file system access
   - Dependent on browser capabilities

2. Native Platforms (iOS/Android/macOS/Linux/Windows)
   - Audio requires temporary file creation
   - File system permissions may be required
   - Platform-specific build requirements

3. Mobile Devices
   - Screen size constraints for complex UI elements
   - Touch interaction requirements
   - Battery and performance considerations

FUNCTIONALITY LIMITATIONS
--------------------------
1. Password Security
   - Passwords are hashed using bcryptjs (10 rounds)
   - No password strength requirements enforced
   - No password expiration or rotation

2. Authentication
   - No JWT token-based session management
   - Limited session timeout handling
   - No multi-factor authentication

3. Data Export
   - CSV export only (no other formats)
   - Limited data filtering options in export

4. Character Interactions
   - Fixed set of characters (not dynamically expandable)
   - Character audio requires pre-upload to database

5. Parent Features
   - No real-time notifications
   - Limited data analysis tools
   - No data comparison between children

6. Games
   - Limited number of games available

7. Tutorial
   - Tutorial may not adapt to user's previous experience
   - No skip functionality for advanced users

TECHNICAL LIMITATIONS
---------------------
1. Database
   - Character images stored as binary (may impact performance)
   - Limited query optimization

2. API
   - GraphQL endpoint is single instance (no load balancing)
   - No rate limiting implemented
   - Limited error recovery mechanisms

3. Deployment
   - Frontend and backend are separately deployed
   - Requires manual coordination for updates

4. Scalability
   - Single database instance
   - No caching layer
   - No CDN for static assets
   - Limited horizontal scaling options
